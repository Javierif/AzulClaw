# Desktop Interface Design

Last reviewed: 2026-04-15

## Product goal

AzulClaw should feel like a capable local companion, not a generic SaaS chat window. The desktop app must make identity, memory, workspace, and automation visible without overwhelming the user.

## Core UX principles

### Chat is the center, not the whole product

Conversation is the primary interaction mode, but the user also needs clear surfaces for:

- workspace
- memory
- processes
- heartbeats
- settings

### Security must be understandable

Users should immediately understand that AzulClaw operates inside a dedicated sandbox and does not have open-ended access to the machine.

### Operational transparency matters

When the assistant is thinking, narrating, or waiting on a slower lane, the interface should explain that state without becoming noisy.

### Strong product identity

The mascot, onboarding language, and visual system should support the idea of a living local companion while staying technically credible.

## Information architecture

Primary navigation:

- Chat
- Skills
- Processes
- Heartbeats
- Memory
- Workspace
- Settings

Secondary context:

- active session title
- workspace path
- top-bar status and live thought indicators

## Visual direction

- deep blue base tones
- brighter cyan or teal accents for operational states
- warm neutrals to avoid a cold enterprise look
- rounded surfaces and atmospheric backgrounds rather than flat utility panels
